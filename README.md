
## 캐시 전략
### 1. 전체 쿠폰 조회  
> 캐시 이름 : COUPON_LIST,  
> 만료 전략 => 
> TTL : 20분  
> 사용자 쿠폰 발급 후 잔여 재고 0일 시 EVICTION

 ### 2. 인기상품 조회
> 캐시 이름 : POPULAR_PRODUCTS_LIST,  
> 만료 전략 =>
> TTL : 24시간 2분  
> 스케줄러에 의해 매 일 00시에 DB를 조회하여 새롭게 캐싱 처리 => Cache Stampede 방지


 ### 3. 상품 상세 조회
> 캐시 이름 : PRODUCT_DETAIL,  
> 만료 전략 =>
> TTL : 10분  
> 사용자 주문에 따른 재고가 20개 미만일 시 캐시 삭제 , 갱신x

### 3. 상품 리스트 조회
> 캐시 이름 : PRODUCT_LIST_ONE_PAGE,  
> 만료 전략 =>
> TTL : 10분  
> 페이징에 따른 4페이지 이하에 대해서만 캐싱 적용,   
> => 이를 통해 메모리 효율성 및 캐시 히트율 최적화.  
> 별도의 eviction x


##  캐시 성능 개선 전후 비교

```
인기 상품 조회:
  평균: 245ms

상품 리스트 조회:
  평균: 180ms
```

### 캐시 적용 후
```
인기 상품 조회:
  평균: 35ms  약 86% 개선

상품 리스트 조회:
  평균: 42ms  약 77% 개선
```

### 후기
실제 캐시의 삭제 및 만료 등에 대한 테스트까지는 하지 못하였습니다.
다만 캐시를 사용함으로서 응답이 훨씬 빨라질 수 있음을 할 수 있었습니다.
