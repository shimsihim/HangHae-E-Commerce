## 💬 **리뷰 요청 사항**

### 질문/고민 포인트
1. 기존에는 컨트롤러와 서비스간에 dto를 전달하지 않고 웹계층의 dto로 직접 전달을 하였습니다.
  이번에 각 계층별로 dto를 만들었는데 단순 객체 반환의 경우 괜찮지만 상품-상품옵션들 과 같이 <br>
  [{ 상품id : id , 상품옵션s : [ ...]},{ 상품id : id , 상품옵션s : [ ...]}]<br>
  의 경우 매핑 코드가 지저분하다는 느낌을 받았습니다.
  QNA시간에 MapStruct 를 권장하지 않는다고 하셨는데, 코치님만의 우아한? 방식이 있을까요?


2. 기존에는 도메인별로 각각 1개의 서비스만을 구현했었습니다.
이번에는 각각의 useCase 별로 어플리케이션 서비스를 만들고 
도메인별로 1개씩 도메인서비스를 구현하였습니다.
위에 대해 다음과 같은 사항이 궁금합니다.
<br>
- useCase별로 나누니 마치 해당 useCase들이 fasade의 역할을 한다고 느껴졌는데 이게 맞는걸까요?
- 주문기능을 예시로 들면 주문하기 useCase에서(어플리케이션 서비스) 쿠폰사용서비스 호출(쿠폰 도메인 서비스), 포인트사용 서비스 호출(유저 포인트 서비스) 등
